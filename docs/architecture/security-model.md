# Security Model

Roche follows a **deny-by-default** security model designed specifically for AI agent workloads.

## Threat Model

AI agents execute code generated by LLMs — fundamentally untrusted input. Common threats:

| Threat | Attack Vector | Roche Mitigation |
|--------|--------------|-----------------|
| Data exfiltration | Network requests to attacker-controlled servers | Network disabled by default |
| Persistent compromise | Writing backdoors to the filesystem | Readonly filesystem by default |
| Resource exhaustion | Infinite loops, fork bombs | Timeout + PID limit |
| Privilege escalation | setuid/setgid binaries | `no-new-privileges` security option |
| Host escape | Container breakout | Minimal image, no capabilities |

## Default Security Configuration

| Setting | Default | Flag to Override |
|---------|---------|-----------------|
| Network | **disabled** | `--network` |
| Filesystem | **readonly** | `--writable` |
| Timeout | **300 seconds** | `--timeout <secs>` |
| PID limit | **256** | — |
| Privileges | **no-new-privileges** | — |
| Image | `python:3.12-slim` | `--image <img>` |

## Design Principles

### Explicit Opt-In

Every dangerous capability requires an explicit flag. There are no "smart defaults" that guess what the agent needs:

```bash
# Safe — defaults apply
roche create

# Operator explicitly grants network
roche create --network

# Operator explicitly grants writable FS
roche create --writable
```

### Defense in Depth

Multiple independent security boundaries:

1. **Network isolation** — `--network none` blocks all outbound/inbound traffic
2. **Filesystem isolation** — `--read-only` prevents writes to the root filesystem
3. **Resource limits** — memory, CPU, PID limits prevent resource abuse
4. **Privilege restrictions** — `no-new-privileges` prevents escalation
5. **Timeout enforcement** — auto-destroy after the configured timeout

### Minimal Attack Surface

- Default image (`python:3.12-slim`) has minimal packages installed
- No capabilities granted to the container
- No host volumes mounted by default

## Comparison with Alternatives

| Aspect | Roche | E2B | Direct Docker | Modal |
|--------|-------|-----|---------------|-------|
| Default network | **off** | on | on | on |
| Default filesystem | **readonly** | writable | writable | writable |
| Default timeout | **300s** | none | none | varies |
| Provider lock-in | None | Firecracker only | Docker only | Modal only |
| Local-first | **Yes** | No (cloud) | Yes | No (cloud) |

## Two Security Dimensions

Roche handles **physical isolation** (what system resources code can access). For **logical security** (what tools an agent can call, budget limits, human approval), see [Castor](https://substratum-labs.github.io/castor-docs/).

| Dimension | Tool | Controls |
|-----------|------|----------|
| Physical | **Roche** | Network, filesystem, CPU, memory, PIDs |
| Logical | **Castor** | Tool access, budgets, HITL approval |

Both are needed for comprehensive AI agent security. They compose independently.
