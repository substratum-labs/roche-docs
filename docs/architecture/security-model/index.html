Security Model

Roche follows a deny-by-default security model designed specifically for AI agent workloads.

Threat Model​

AI agents execute code generated by LLMs — fundamentally untrusted input. Common threats:

ThreatAttack VectorRoche Mitigation
Data exfiltrationNetwork requests to attacker-controlled serversNetwork disabled by default
Persistent compromiseWriting backdoors to the filesystemReadonly filesystem by default
Resource exhaustionInfinite loops, fork bombsTimeout + PID limit
Privilege escalationsetuid/setgid binariesno-new-privileges security option
Host escapeContainer breakoutMinimal image, no capabilities

Default Security Configuration​

SettingDefaultFlag to Override
Networkdisabled--network
Filesystemreadonly--writable
Timeout300 seconds--timeout <secs>
PID limit256—

Design Principles​

Explicit Opt-In​

Every dangerous capability requires an explicit flag. There are no "smart defaults" that guess what the agent needs:

# Safe — defaults apply
roche create

# Operator explicitly grants network
roche create --network

# Operator explicitly grants writable FS
roche create --writable

Defense in Depth​

Multiple independent security boundaries:

  1. Network isolation — --network none blocks all outbound/inbound traffic
  2. Filesystem isolation — --read-only prevents writes to the root filesystem
  3. Resource limits — memory, CPU, PID limits prevent resource abuse
  4. Privilege restrictions — no-new-privileges prevents escalation
  5. Timeout enforcement — auto-destroy after the configured timeout

Minimal Attack Surface​

  • Default image (python:3.12-slim) has minimal packages installed
  • No capabilities granted to the container
  • No host volumes mounted by default

Two Security Dimensions​

Roche handles physical isolation (what system resources code can access). For logical security (what tools an agent can call, budget limits, human approval), see Castor.

DimensionToolControls
PhysicalRocheNetwork, filesystem, CPU, memory, PIDs
LogicalCastorTool access, budgets, HITL approval

Both are needed for comprehensive AI agent security. They compose independently.
